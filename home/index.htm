HyperText Markup Language — «язык гипертекстовой разметки») — стандартизированный язык гипертекстовой разметки документов для просмотра веб-страниц в браузере. Веб-браузеры получают HTML документ от сервера по протоколам HTTP/HTTPS или открывают с локального диска, далее интерпретируют код в интерфейс, который будет отображаться на экране монитора Элементы HTML являются строительными блоками HTML страниц. С помощью HTML разные конструкции, изображения и другие объекты, такие как интерактивная веб-форма, могут быть встроены в отображаемую страницу. HTML предоставляет средства для создания заголовков, абзацев, списков, ссылок, цитат и других элементов. Элементы HTML выделяются тегами, записанными с использованием угловых скобок. Такие теги, как img/ и input />, напрямую вводят контент на страницу. Другие теги, такие как , окружают и оформляют текст внутри себя и могут включать другие теги в качестве подэлементов. Браузеры не отображают HTML-теги, но используют их для интерпретации содержимого страницы Язык XHTML является более строгим вариантом HTML, он следует синтаксису XML и является приложением языка XMLв области разметки гипертекста. В HTML можно встроить программный код на языке программирования JavaScript, для управления поведением и содержанием веб-страниц. Также включение CSS в HTML описывает внешний вид и макет страницы.

CSS ( англ. Cascading Style Sheets ) — язык формального описания, написанный с использованием языка разметки внешнего вида документа . Обычно он используется для описания и компоновки веб-страниц, написанных на языках разметки HTML и XHTML , но также может применяться к любому другому XML-документу, например SVG или XU При создании HTML-документов веб-дизайнеры сталкиваются с множеством сложных вопросов: смена паролей в пределах определенного абзаца; размещение текста или изображения в любом месте документа; изменение цвета текста или фона; форматирование различных таблиц данных и т.д. Эти задачи также можно выполнять стандартным образом, то есть есть возможность описать каждый абзац на языке HTML отдельным набором команд. В этом случае процесс написания программы станет слишком сложным, а размер документа будет слишком большим. Для упрощения этой задачи к страницам прикрепляется внешний файл, реализованный в стандарте CSS, и страница один раз форматируется с использованием специального макроязыка, используемого в этом файле. Иными словами, в качестве шаблона здесь используется файл CSS, с помощью которого форматируются тексты, таблицы и другие элементы в HTML-документах. Стандарт CSS можно использовать на любом сервере без каких-либо ограничений . Подобно тому, как коды CSS могут быть записаны в коды HTML , операция также может быть создана путем создания внешнего файла CSS. В зависимости от типа их можно купить в разделах cod BODY/ code> или code HEAD code. В качестве альтернативы рабочий лист файла CSS можно добавить с помощью . Хранение их в отдельном файле или файлах гарантирует, что их можно будет использовать на разных HTML -страницах, что упрощает форматирование одной и той же страницы для разных устройств. Добавление кодов CSS в файл HTML можно выполнить тремя различными способами. В стандарте CSS файлы имеют формат .css .

JavaScript (; аббр. JS ɛs./) — мультипарадигменный язык программирования. Поддерживает объектно-ориентированный, императивный и функциональный стили. Является реализацией спецификации ECMAScript (стандарт ECMA-262[9]). JavaScript обычно используется как встраиваемый язык для программного доступа к объектам приложений. Наиболее широкое применение находит в браузерах как язык сценариев для придания интерактивности веб-страницам[10]. Основные архитектурные черты: динамическая типизация, слабая типизация, автоматическое управление памятью, прототипное программирование, функции как объекты первого класса. На JavaScript оказали влияние многие языки, при разработке была цель сделать язык похожим на Java. Языком JavaScript не владеет какая-либо компания или организация, что отличает его от ряда языков программирования, используемых в веб-разработке[~ 1][11]. Название «JavaScript» является зарегистрированным товарным знаком корпорации Oracle в США[12].

JSON (англ. JavaScript Object Notation, обычно произносится как ən/ JAY-sən[3]) — текстовый формат обмена данными, основанный на JavaScript. Как и многие другие текстовые форматы, JSON легко читается людьми. Формат JSON был разработан Дугласом Крокфордом[4]. Несмотря на происхождение от JavaScript (точнее, от подмножества языка стандарта ECMA-262 1999 года), формат считается независимым от языка и может использоваться практически с любым языком программирования. Для многих языков существует готовый код для создания и обработки данных в формате JSON.За счёт своей лаконичности по сравнению с XML формат JSON может быть более подходящим для сериализации сложных структур. Применяется в веб-приложениях как для обмена данными между браузером и сервером (AJAX), так и между серверами (программные HTTP-сопряжения). Поскольку формат JSON является подмножеством синтаксиса языка JavaScript, то он может быть быстро десериализован встроенной функцией JSON.parse().
